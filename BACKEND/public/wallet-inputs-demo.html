üìñ How It Works

NAKAMA automatically detects and populates wallet address inputs on your site using the user's registered addresses.

// Basic usage - auto-populate all detected wallet inputs NAKAMA.populateInputs(); // Advanced usage with custom options NAKAMA.populateInputs({ selectors: { solana: ['input[name="solana-wallet"]'], ethereum: ['input[name="eth-address"]'] }, chainPriority: ['solana', 'ethereum', 'polygon'], autoDetect: true }); // Watch for new inputs (useful for dynamic content) const watcher = NAKAMA.watchInputs({ interval: 1000, maxAttempts: 60 }); // Stop watching NAKAMA.stopWatching(watcher);

‚ö° Auto-Triggering

The widget automatically triggers form submissions and button clicks after populating inputs:

// Configure auto-triggering NAKAMA.loginWidget.options = { autoTrigger: true, // Enable auto-triggering (default: true) autoTriggerDelay: 500 // Delay before triggering (default: 500ms) }; // The widget will automatically: // 1. Populate wallet address inputs // 2. Wait for the configured delay // 3. Look for submit buttons or forms // 4. Click buttons or submit forms automatically

What Gets Triggered:

  • Form Submissions: Submit buttons in forms containing wallet inputs
  • Nearby Buttons: Action buttons near the populated inputs
  • Common Actions: Buttons with "search", "analyze", "load", "fetch" text
  • Wallet-Related: Buttons mentioning "wallet", "address", "portfolio"
